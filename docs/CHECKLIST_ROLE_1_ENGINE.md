# Webster: чеклист роли 1 (движок Canvas)

Ты работаешь здесь над canvas-частью редактора и проверяешь ее на отдельной странице фронта. Основной цикл работы такой: запускаешь frontend через `pnpm dev` или через Docker, открываешь страницу `/canvas-engine`, и уже там тестируешь поведение объектов, выделение, перемещение, resize, rotate и историю действий. Если меняешь логику движка, сразу смотри результат на этой странице, а не только по логам.

Для этой роли не нужен финальный UX/UI слой: это зона ответственности роли 2. Тебе достаточно временного технического интерфейса (кнопки, hotkeys, debug-панель) на странице `/canvas-engine`, чтобы быстро валидировать API движка и корректность сценариев.

Конечный результат должен быть таким: отдельная тестовая страница стабильно открывается в браузере, на ней можно вручную проверять все базовые сценарии движка, а изменения в коде либо видны сразу в dev-режиме, либо после перезапуска контейнера/процесса в зависимости от способа запуска.

Смотреть тест движка здесь: http://localhost:5173/canvas-engine

Цель роли: реализовать собственный canvas-движок (рендер, модель сцены, математика, сериализация, команды).
